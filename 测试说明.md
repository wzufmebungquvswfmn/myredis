# 测试说明

## 性能测试

**目的与方法**  
使用 `redis-benchmark` 在 16 个场景中对比官方 Redis（6379）与 myredis（6380）的吞吐性能。场景包括：基础 SET/GET、Pipeline、并发、随机 key、混合命令等。

**运行**
```bash
./scripts/run_benchmark.sh          # 快速模式（默认）
./scripts/run_benchmark.sh --full   # 完整模式
```

**输出结构**
```
benchmark_logs/run_时间戳/
├── summary.md                      # 汇总表格
├── run.log                         # 完整日志
└── raw/s{1-16}_p{6379,6380}.log   # 原始输出
```

**如何查看结果**  
打开 `summary.md` 查看汇总表：
- 每行对应一个场景
- 数据格式为"SET xxx rps ; GET yyy rps"（两个端口分别显示）
- `比例(%)` 列表示性能差异（100% = 相同，>100% = myredis 更快）
- `⚠` 标记表示异常（SET/GET 吞吐差异过大）

## 功能与网络测试

**目的与方法**  
- 功能测试：验证 Redis 协议正确性（26 个单元/库测试）
- 网络集成测试：验证并发连接、多客户端交互（26 个集成测试）

**运行**
```bash
./scripts/run_test.sh
```

**输出结构**
```
test_logs/run_时间戳/
├── summary.md                      # 汇总表格
├── run.log                         # 完整日志
└── raw/
    ├── functional.log              # 单元测试输出
    └── network_integration.log     # 集成测试输出
```

**如何查看结果**  
打开 `summary.md` 查看两个测试套件的结果：
- `通过数` 和 `失败数` 列，理想情况为"26 / 0"
- 失败的测试会有链接指向 raw 日志查看详细错误
